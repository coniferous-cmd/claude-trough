# UI

`trough` has two interfaces: command mode and list mode.

## Command Mode

Command mode is used when `trough` receives a subcommand. It is optimized for quick shell usage:

```text
trough add "task"
trough list
trough done 1
```

Command mode should print concise, script-friendly output.

`trough push` prints no output on success. `trough list` prints no output when
there are no active tasks.

## List Mode

List mode is the default when running `trough` without arguments. It opens an interactive terminal UI.

Keyboard controls:

- `j` or `Down`: move selection down.
- `k` or `Up`: move selection up.
- `Space`: toggle the selected task.
- `Enter`: edit selected task detail.
- `q`: quit.

## Display Rules

- CLI incomplete tasks display as `❌`.
- CLI completed tasks display as `✅`.
- Nonzero priority displays as `P<n>`.
- Completed tasks are styled differently in the TUI.
- Task detail is edited through the external editor but is not shown in the list.

## Editor Behavior

When editing details, `trough` opens the user's editor. Editor selection order:

1. `$EDITOR`
2. `$VISUAL`
3. `vi`

The TUI must leave raw/alternate-screen mode before launching the editor, then restore the TUI after editing.
